# Accelerating Distributed Deep Learning using Multi-Path RDMA in Data Center Networks

<!-- id=238780460 parent=238780418 -->

## Overview

Data center networks (DCNs) widely deploy RDMA to support data-intensive applications, including machine learning, but current RDMA hardware lacks multi-path transport support. This paper presents Maestro, a purely software-based multi-path RDMA solution to effectively utilize the rich multi-path topology in DCNs. By addressing the limitation of single-path RDMA, Maestro significantly improves bandwidth utilization, reduces network congestion, and accelerates distributed deep learning training. The proposed solution integrates seamlessly into existing RDMA infrastructures without hardware modifications. Evaluations show that Maestro achieves substantial performance improvements in distributed deep learning workloads compared to baseline RDMA configurations.

## Objective

The objective of this paper is to develop and evaluate Maestro, a software-based multi-path RDMA transport mechanism that enhances distributed deep learning performance by improving bandwidth utilization and reducing network congestion in data center networks.

## Keywords:

- Remote Direct Memory Access (RDMA)

- Data Center Networks (DCNs)

- Distributed Deep Learning (DDL)

- Multi-Path Transport

- Network Congestion and Load Balancing

## As-Is

- Existing RDMA implementations in DCNs only support single-path transport, leading to suboptimal bandwidth utilization and network congestion.

- Single-path RDMA can cause link overloading and inefficient use of available network resources, degrading the performance of distributed deep learning workloads.

- Traditional TCP-based multi-path solutions incur substantial overhead and increased latency due to connection management complexity, packet reordering issues, congestion control inefficiencies, higher acknowledgment delays, and additional CPU/memory load, making them unsuitable for meeting the stringent low-latency requirements of RDMA environments.

- Distributed deep learning workloads generate bursty and high-bandwidth traffic, necessitating an efficient transport mechanism to fully utilize available network paths.

- Previous software-based solutions for multi-path transport require modifications to RDMA hardware, limiting their deployability and practicality in existing infrastructures.

## To-Be

- Maestro is a purely software-based multi-path RDMA solution that enables efficient utilization of available network paths in DCNs by leveraging virtual RNIC (vRNIC) technology to create multiple logical RDMA interfaces, allowing flexible and scalable path selection without modifying physical hardware.

- Maestro dynamically adjusts path selection based on real-time network conditions by utilizing a user-space middleware layer that splits messages across multiple virtual RNIC (vRNIC) instances. This middleware monitors congestion levels, packet loss rates, and link utilization, ensuring efficient traffic distribution across available paths.

- Maestro leverages congestion-aware path selection and adaptive load balancing techniques, utilizing real-time telemetry data and active feedback mechanisms to optimize performance and minimize latency.

- The design of Maestro is fully compatible with existing RDMA hardware, requiring no modifications to network infrastructure. Also, the system seamlessly integrates with distributed deep learning frameworks to accelerate model training by reducing communication bottlenecks.

## Methodology

- The paper implements Maestro as a software extension to existing RDMA stacks, requiring no hardware modifications.

- Evaluations are conducted using real-world distributed deep learning workloads, including ResNet-50 and BERT training on GPU clusters.

- The study compares Maestro against traditional single-path RDMA configurations and existing multi-path solutions to measure bandwidth utilization, latency, and training time.

- Additionally, the network performance is analyzed using congestion metrics, packet loss rates, and throughput measurements.

- The experimental setup includes various network topologies, such as fat-tree, torus, and leaf-spine, to test the scalability and robustness of the proposed solution.

## Results

- Maestro achieves significant performance improvements in distributed deep learning training by enhancing bandwidth utilization and reducing network congestion.

- Compared to single-path RDMA, Maestro reduces communication overhead by up to 40% and speeds up training times by 1.5× to 2.3×.

- The solution demonstrates effective load balancing across multiple paths, preventing bottlenecks and optimizing resource usage.

- Experimental results validate the practicality of software-based multi-path RDMA in real-world data center deployments, achieving up to 95% network bandwidth utilization.

- Future work includes extending Maestro to support additional network architectures, such as Clos and Jellyfish topologies, and refining congestion control mechanisms through enhanced real-time telemetry, such as in-band network telemetry (INT) and software-defined networking (SDN) monitoring and AI-driven adaptive traffic management.
