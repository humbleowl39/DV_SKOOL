### How to run rccl-test?

1.  Load the amdgpu

    1.  `sudo modprobe amdgpu` for both node after reboot

2.  Load the RDMA driver

3.  Check if the rdma device is up correctly with `mb_0`, `mb_1`

    1.  <div>

        <div>

        kanghyun.choi@rdma4:~/test_rdma/scripts\$ rdma link link mlx5_0/1 state DOWN physical_state DISABLED netdev enp189s0f0np0 link mlx5_1/1 state DOWN physical_state DISABLED netdev enp189s0f1np1 link mb_0/1 state ACTIVE physical_state LINK_UP netdev enp99s0f0np0 link mb_1/1 state ACTIVE physical_state LINK_UP netdev enp100s0f0np0

        </div>

        </div>

    2.  If the device name is different from `mb_0`, `mb_1`

        1.  `sudo apt purge rdma-core -y`

        2.  reload the user/kernel driver

4.  Set L3 routing

    1.  rdma4 : run `test_rdma/scripts/gh/r4_l3.sh`

    2.  rdma3 : run `test_rdma/scripts/gh/r3_l3.sh`

5.  **RUN rccl-test**

    1.  **<span style="color: rgb(255,86,48);">\[Notice\] we can only run rccl at rdma4</span>**

    2.  2way `test_rdma/scripts/rccl.sh [op_name]`

    3.  1way `test_rdma/scripts/rccl_1way.sh [op_name]`

    4.  supporting operation types

        1.  all_gather, all_reduce, alltoall, alltoallv , broadcast, gather, reduce, reduce_scatter, scatter, sendrecv

###  Performance (baseline)

- including both out-of-place, in-place bandwidth

- approximately below performance with fluctuations

  - all_reduce ~10GB/s

  - alltoall ~4.8GB/s

- (Advanced) What is difference between [alg bw](http://alg.bw) and bus bw?

  - refer this [Link](https://github.com/ROCm/rccl-tests/blob/develop/doc/PERFORMANCE.md)

  - TL;DR : all we need to consider is **bus bw**
