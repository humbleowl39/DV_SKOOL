# 5. Send interrupt to device driver

<!-- id=471040230 parent=471040007 -->

## Introduction

In this step, you modify calculator engine and device driver to send an interrupt so that the driver can recognize the received result. You can verify the result by checking whether the result is printed even if the driver doesn’t poll the memory. The result shall be as follows.

### Input

<div>

<div>

./my_calculator 13 21

</div>

</div>

### Output

<div>

<div>

\$ dmesg ... \[33517.137631\] qdma_driver: write 13 21 \[33517.139631\] qdma_driver_interrupt: result = 34

</div>

</div>

## Hint

### Interrupt handler

Interrupt handler is a special block of code associated with a specific [i](https://en.wikipedia.org/wiki/Interrupt)nterrupt condition. You can modify interrupt handler code in QDMA device driver. Please refer to the materials at the link below or other online resources.
