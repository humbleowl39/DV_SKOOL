# DiffServ and Traffic Class

<!-- id=229998657 parent=229998593 -->

This is a background section to understand concepts in the PFC description.

Differentiated Services (DiffServ) is a network architecture that enhances Quality of Service (QoS) by classifying and managing traffic into distinct classes, each receiving specific forwarding treatments.

### Traffic Classification in DiffServ ([IEEE P802.1p](https://en.wikipedia.org/wiki/IEEE_P802.1p))

This section introduces a brief summary of the DSCP field. There is a TC field of 8 bits in the IP header, and we utilize 3 MSBs of the TC field to define a traffic class with 8 priorities

In DiffServ, traffic entering the network is classified and marked at the boundaries (edge routers) based on predefined policies. This classification uses the Differentiated Services Code Point (DSCP), a 6-bit value in the IP header (8 bits but 2 LSBs are not used), to indicate the priority and handling requirements of each packet. The DSCP value determines the Per-Hop Behavior (PHB) that each router along the packet's path applies, ensuring consistent treatment across the network.

Eight different classes of service are available as expressed through the 3-bit PCP field in an IEEE 802.1Q header added to the frame. The way traffic is treated when assigned to any particular class is undefined and left to the implementation. The IEEE, however, has made some broad recommendations.

This concept is used in the type of service (ToS) field which is the second byte of the IPv4 header. It has had various purposes over the years, and has been defined in different ways by five [RFCs](https://en.wikipedia.org/wiki/Request_for_Comments).

The modern redefinition of the ToS field (as well as the Traffic Class field in IPv6 packets) splits this byte into a 6-bit Differentiated Services (DS) field and a 2-bit Explicit Congestion Notification (ECN) field. While Differentiated Services is somewhat backwards compatible with ToS, ECN is not.

### Common PHBs and Their DSCP Values:

</div>

**Default Forwarding (DF):** Represents best-effort traffic with no special QoS requirements. Note that the underlined bits are fixed. DSCP value: <u>000000</u>.

**Class Selector (CS):** Maintains backward compatibility with the IP precedence field, with values ranging from CS0 to CS7, corresponding to DSCP values 000<u>000</u> to 111<u>000</u>.

**Assured Forwarding (AF):** Provides delivery assurance under specified conditions, with four classes (AF1 to AF4) and three drop precedences (low, medium, high) within each class, resulting in twelve possible DSCP values (e.g., AF11, AF12, AF43 with DSCP values 00101<u>0</u>, 00110<u>0</u>, and 10011<u>0</u>).

**Expedited Forwarding (EF):** Designed for low-latency, low-loss, and low-jitter services, suitable for real-time applications like VoIP. DSCP value: <u>101110</u>.
