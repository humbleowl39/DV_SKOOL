# NetReduce: RDMA-Compatible In-Network Reduction for Distributed DNN Training Acceleration

<!-- id=240484911 parent=240550447 -->

## Overview

NetReduce is a novel RDMA-compatible in-network reduction architecture designed to accelerate distributed deep neural network (DNN) training by integrating in-network computing with Remote Direct Memory Access (RDMA). Unlike existing solutions, NetReduce does not terminate connections in the network but instead maintains a reliable connection between end-hosts, thereby fully leveraging congestion control and reliability features of RoCE. By avoiding complex network protocol stacks within switches, NetReduce reduces hardware costs while maintaining efficiency. The system is designed to function without modifying commodity hardware such as Network Interface Cards (NICs) or switches, making it an easily deployable solution. The FPGA-based prototype demonstrates significant performance improvements in distributed DNN training.

## Objective

NetReduce aims to accelerate distributed DNN training by introducing an RDMA-compatible in-network reduction mechanism, improving all-reduce operation performance while maintaining reliability, congestion control, and compatibility with existing RoCE-based infrastructures.

## Background

RDMA, RoCE, In-network computing, Distributed deep learning, All-reduce operation, FPGA-based acceleration, Congestion control in high-speed networks

## As-Is

- Existing in-network solutions require connection termination in the network, which complicates congestion control and reliability management.

- RoCE-based solutions lack efficient in-network computing mechanisms, leading to substantial delays in data aggregation, increased communication overhead, and underutilization of available network bandwidth. These inefficiencies cause bottlenecks in distributed DNN training, limiting scalability and overall system performance.

- Traditional approaches face significant challenges in simultaneously optimizing efficiency, reliability, and ease of deployment. These methods often require extensive modifications to network devices, including specialized firmware updates, hardware reconfigurations, and custom network protocols, increasing both complexity and cost. Additionally, the lack of standardization in in-network computing mechanisms further exacerbates deployment challenges.

- Congestion control and reliability in RoCE environments are not optimized for in-network reductions, leading to performance inefficiencies in distributed training.

- Observations indicate that offloading all-reduce operations to an in-network device without terminating RDMA connections can significantly improve performance.

## To-Be

- NetReduce enables in-network reduction without terminating RDMA connections, preserving the reliability and congestion control benefits of RoCE.

- The system fully reuses existing congestion control mechanisms, avoiding the need for complex modifications to NICs or switches.

- It employs an FPGA-based in-network computing module that offloads reduction operations from end-hosts to the network switch, enabling real-time data aggregation. This mechanism leverages parallel processing capabilities within the FPGA to accelerate reductions while preserving RDMA semantics, reducing network congestion and improving overall system efficiency.

- A lightweight control mechanism is introduced, enabling the switch to dynamically coordinate with end-hosts for optimized all-reduce operations. This mechanism leverages metadata tagging and efficient flow control techniques to minimize synchronization delays and ensure seamless integration with existing network protocols, without necessitating extensive hardware modifications.

- The architecture ensures minimal deployment overhead, making it an out-of-the-box solution that does not require changes to existing network hardware.

## Methodology

- An FPGA-based prototype was implemented, utilizing an Intel Stratix 10 FPGA with 10GbE network interfaces to offload and accelerate in-network reduction operations. The prototype features a programmable data pipeline with hardware-accelerated floating-point operations, optimized for low-latency aggregation. It also incorporates a custom RoCEv2-compatible transport layer to ensure seamless integration with RDMA-based communication protocols.

- Experimental setup included RoCE-enabled RDMA networks, commonly used in distributed DNN training clusters.

- Performance was compared against conventional software-based all-reduce operations and existing RDMA-based in-network solutions.

- Metrics such as latency, bandwidth utilization, and end-to-end training performance were measured to validate the proposed design.

- Traffic patterns from real-world DNN training workloads were used to assess the practical benefits of NetReduce.

## Results

- NetReduce reduces all-reduce latency by up to 42%, leading to improved efficiency in distributed DNN training.

- Bandwidth utilization is optimized, reducing congestion by approximately 35% and improving overall network performance.

- Compared to software-based all-reduce solutions, NetReduce demonstrates a 30% reduction in training time.

- The proposed FPGA-based in-network computing mechanism achieves acceleration without requiring modifications to existing hardware.

- Future research can explore extending NetReduce to support additional in-network operations beyond all-reduce, further improving distributed computing efficiency.
