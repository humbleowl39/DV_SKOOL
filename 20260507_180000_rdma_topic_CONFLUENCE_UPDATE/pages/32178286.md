# RDMA one-sided operation

<!-- id=32178286 parent=32178213 -->

RDMA one-sided operation is a RDMA operation that exploits shared-memory abstraction, in order to remove synchornization overhead existing in traditional RPC-style communcations. That is, only the initiator side’s CPU is involved in the entire RDMA operation process. This is enabled with specialized hardware named RNIC, and server-side RNIC performs DMA on behalf of server-side CPU (which enables much lower latency compared to traditional network stacks).

RDMA Read and RDMA Write are one-sided RDMA primitives. In RDMA Read, the receiver will read the data from the sending node by posting an RDMA

<div>

Read request into the Send Queue

</div>

. The request specifies the remote address of the data (= remote address of MR) to read from and a local buffer to store the data into. The sender remains completely passive in the communication.

<div>

The local network adapter

</div>

asynchronously performs the remote read operation, populates the corresponding buffers, and

<div>

posts a completion event

</div>

to the local Completion Queue when the operation has been completed. In RDMA Write, the sender posts an RDMA Write request to write data to the receiving node. The request specifies the local buffer, which contains the data to be written to the receiver and the remote address that the data will be written to.

- RDMA write
