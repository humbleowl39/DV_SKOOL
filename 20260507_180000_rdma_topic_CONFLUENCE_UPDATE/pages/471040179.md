# 2. Build a device driver

<!-- id=471040179 parent=471040007 -->

## Introduction

In this step, you build a device driver that passes operands to calculator engine. You can verify the result by reading passed operands back from the engine, then printing them at the device driver. The result shall be as follows.

### Input

<div>

<div>

./my_calculator 13 21

</div>

</div>

### Output

<div>

<div>

\$ dmesg ... \[33517.137631\] qdma_driver: write 13 21 \[33517.137631\] qdma_driver: read 13 21

</div>

</div>

## Hint

### MMIO

MMIO (Memory Mapped Input/Output) is a mechanism to access IO devices via memory-like semantics. You should modify calculator engine to allow MMIO and device driver to access memory-mapped address space. Please refer to the materials at the link below or other online resources.

<a href="https://www.kernel.org/doc/html/latest/driver-api/device-io.html?highlight=ioremap#memory-mapped-io" data-card-appearance="inline">https://www.kernel.org/doc/html/latest/driver-api/device-io.html?highlight=ioremap#memory-mapped-io</a>

### PCIe BAR

PCIe specification defines PCIe BAR (Base Address Register). It defines the memory space software can access through PCIe protocol. Please refer to the materials at the link below or other online resources.

<a href="https://stackoverflow.com/questions/30190050/what-is-the-base-address-register-bar-in-pcie" data-card-appearance="inline">https://stackoverflow.com/questions/30190050/what-is-the-base-address-register-bar-in-pcie</a>
