## 4. Code Change Analysis

### 4.1 responder_frontend (SKRP-782: recv local protection error handling)

**Files changed:** `responder_frontend_rx.cpp`, `responder_state_util.cpp`, `responder_frontend.h`

**Changes by thread:**

- **th_mr_rd:** Added `is_invalid_request` early exit (skips MR read), send vs write/read MR-invalid error differentiation (different CQ targets: `recv_cqn` vs `RSVD_CQ_ERR`), blank MR on error. Ops +70%.
- **th_psn_check:** Added `is_invalid_request` branch to push `m_drop2=1` (payload drop coordination with th_dma_write).
- **th_prot_check:** Fully implemented SEND lkey/PD/access/range protection check (was "not supported yet"). Shared comparator optimization: mux inputs to single comparator set (RETH vs SEND paths are mutually exclusive). 4-way ternary error priority encoding.
- **th_dma_write:** Added `is_invalid_request` early exit (skips scatter). `scatter_send_desc` now validates 2nd SGE MR on SGE boundary crossing with full lkey/PD/access/range check. New error port 3 for th_dma_write errors.
- **Error mux:** Expanded from 3 to 4 inputs (new port for th_dma_write).

**Timing-critical paths identified:**

1.  **th_prot_check shared comparator MUX (lines 661-674):** 4x ternary MUX (24/64-bit wide) feeding comparators creates `MUX → CMP` serial chain
2.  **scatter_send_desc 2nd SGE range check (lines 320-341):** 64-bit subtraction + comparison inside while(payload_len) loop
3.  **th_prot_check send error ternary cascade (lines 721-745):** 4-way priority encoding

### 4.2 completer_frontend (SKRP-797: requester/completer local protection error handling)

**Files changed:** `completer_frontend.cpp`, `completer_frontend.h`, `completer_frontend_ccore.h` (NEW), `completer_frontend_def.h`, `completer_state_util.cpp`

**Changes by thread:**

- **th_comp:** Added error early-exit (frozen epsn, skip MODIFY on error). This *improved* latency (108→52) and throughput (109→53).
- **th_dma_write (major regression):** Added multi-SGE MR pre-validation before scatter:
  - Phase 0: Pop initial MR + combinational validation via `check_local_mr_violation()` ccore
  - Same-lkey check: Unrolled combinational loop across all SGEs
  - Phase 1 (II=1): Pipelined AWD pushes for unique lkey transitions to `m_mr_rw_port_scatter`
  - Phase 2 (II=1): Pipelined RD pops + validation for each unique lkey
  - Error path: payload drop + `send_err_completion_rdma_trimmed()`

  This sequential Phase 1 → Phase 2 dependency caused Latency 8→29, Throughput 10→32.
- **Error mux:** Expanded from 3 to 4 inputs (new port for th_dma_write MR violations).

**Correctness constraint:** All SGE MRs must be validated *before* any DMA write to prevent partial writes with invalid MRs (data integrity).
