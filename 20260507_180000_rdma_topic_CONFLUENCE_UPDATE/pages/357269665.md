# Fifo optimization

<!-- id=357269665 parent=7012616 -->

Multiple Fifos are placed between many modules. While sometimes the fifos are necessary, some fifos are not. Let’s get into the difference and understand the reasons why this fifo is required. This is repeatedly important when we are faced resource optimization for BRAM/URAMs. Moreover, the importance is increasing since the data-width should be increased to achieve higher performance.

We can understand Fifos in RDMA_TOP with five regions

\(1\) Fifos for TX CMD path

\(2\) Fifos for TX DATA path

\(3\) Fifos for RX CMD path

\(4\) Fifos for RX DATA path

\(5\) Fifos for descriptors
