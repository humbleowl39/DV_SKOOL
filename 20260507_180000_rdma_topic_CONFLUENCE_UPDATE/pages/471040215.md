# 4. Send result to device driver

<!-- id=471040215 parent=471040007 -->

## Introduction

In this step, you modify calculator engine and kernel driver to send the result to the driver. You can verify the result by reading memory a few seconds after passing operands. The result shall be as follows.

### Input

<div>

<div>

./my_calculator 13 21

</div>

</div>

### Output

<div>

<div>

\$ dmesg ... \[33517.137631\] qdma_driver: write 13 21 \[33517.137631\] qdma_driver: wait... \[33517.139631\] qdma_driver: \*mem = 34

</div>

</div>

## Hint

### DMA

Direct memory access (DMA) is a technology that allows hardware devices to transfer data between themselves and memory without involving CPU. In this step, you should allocate a buffer in software memory to be used in DMA. This can be done by function `dma_alloc_coherent`. (Why is the name “coherent”?) Throughout the step, you need to carefully decide whether to use virtual address or physical address. Please refer to the materials at the link below or other online resources.
