## Introduction

In this step, you build a user application that receives user input and passes them to device driver. You can verify the result by printing the arguments at the device driver. The result shall be as follows.

### Input

<div>

<div>

./my_calculator 13 21

</div>

</div>

### Output

<div>

<div>

\$ dmesg ... \[33517.137631\] qdma_driver: 13 21

</div>

</div>

## Hint

### Device driver: QDMA

We use the QDMA driver provided by Xilinx. You need to modify this driver to complete the task. You can get the code from Xilinx github. <a href="https://github.com/Xilinx/dma_ip_drivers" data-card-appearance="inline">https://github.com/Xilinx/dma_ip_drivers</a>

Because we use Vivado 2020.2, you should use QDMA driver version 2020.2 as well. Checkout branch by typing the command below.

<div>

<div>

git checkout 2020.2

</div>

</div>

You can build your own device driver by running the command below. We use `qdma-pf.ko` kernel module as a device driver.

<div>

<div>

~/dma_ip_drivers/QDMA/linux-kernel\$ make driver ~/dma_ip_drivers/QDMA/linux-kernel\$ ls ./driver/src/qdma-pf.ko qdma-pf.ko

</div>

</div>

### IOCTL

You can use `ioctl` system call to control hardware. `ioctl` is a system call for device-specific input/output operations and other operations which cannot be expressed by regular file semantics. Please refer to the materials at the link below or other online resources.

<a href="https://man7.org/linux/man-pages/man2/ioctl.2.html" data-card-appearance="inline">https://man7.org/linux/man-pages/man2/ioctl.2.html</a>
