Disclaimer. There were some conflicts between *IBTA Specification 1.7* vs. *NVIDIA patent + experimental result*. Regarding such case, the information from the latter is depicted here.

MAD packet = ETH header + IP header + UDP header + BTH + DETH + Payload (= MAD data format)

- BTH : opcode = 0x64 (UD send), dst QP = 0x01

- DETH : Pkey = 0x8001, src QP = 0x01

  - Method = RTTProbe uses only **0x01** (Get)

  - R = Response bit (set to **1** for CCMAD RTTProbe response)

  - ClassVersion = Reserved to **0x01**

  - MgmtClass = **0x0d** for CCMAD (vender-specific class)

  - BaseVersion = Reserved to **0x01**

  - ClassSpecific = **0x00** unused

  - Status = refer to the figure below

  - TransactionID = In CCMAD, this breaks down into {**Flow Tag \[31:0\]** + **PSN \[55:32\]** + **Reserved 0x00 \[63:56\]**}

  - AdditionalStatus = Unused **0x0000**

  - AttributeID = **0x0010** for RTTProbe

  - AttributeModifier = Unused **0x00000000**

- RTTProbe32 data format

  - <div>

    </div>

  - Requestor Opaque Data = {**Unused 0x00000000 \[31:0\]**, **Sender’s timestamp \[63:32\]**}

    - Timestamp filled when the HW sends out the packet

  - Request Receive Time = **Responder’s timestamp 1**

    - Filled when the HW receives this packet

  - Response Send Time = **Responder’s timestamp 2**

    - Filled when the HW sends out the response

  - Reserved = Unused, but **up to 16 bytes are 0-filled** to make the entire MAD payload (including common MAD header) to be exactly 64B
