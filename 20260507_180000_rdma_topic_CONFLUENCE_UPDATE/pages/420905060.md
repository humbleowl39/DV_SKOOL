**Notes** on `init.sh`

The `init.sh` script performs two tasks:

1.  Run `git submodule update --init` for MB-Shell and its submodules

2.  Sets up environment variables

- You should **source** `init.sh` **the first time** you initialize the MB-Shell repository. This will configure everything correctly.

- Later, if you need to **re-source** `init.sh` **(e.g., when generating a bitfile)**, the environment variables will be set properly.

- However, re-sourcing `init.sh` also re-checks out the RDMA branch to the commit linked with the MB-Shell branch.

  - This can overwrite your changes if you have modified the RDMA code.

**Recommendation**

After the initial setup, leave the submodule update lines **commented out**. This way, you can safely re-source `init.sh` to set environment variables without affecting your RDMA working branch.

## How to update the block design TCL

1.  Generate LP project (no code modifications yet)

2.  Open the project (Vivado 2024.2)

3.  Apply code modification & **replace the modified modules**

    1.  MB-Shell creates projects by **copying all source files** into the build directory

    2.  Therefore, if you want to modify a module, you must **replace it inside the Vivado project**

        1.  In Vivado, go to Sources → \[module\] → right-click → Replace Module

4.  Modify the block design

5.  Export the block design tcl file to any location (do not forget to validate design)

6.  Run `make tcl` script

    1.  A block design tcl must be saved **per device**.

    2.  The `make tcl` reformats the tcl for the correct device and places it in the proper location.

    3.  Current RDMA 2x200G design uses data width **1024**
