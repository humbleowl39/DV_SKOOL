# Summary

**The BDF on the Mi325x datasheet does not match the physical PCIe slot. This post corrects this.**

You can think of this diagram as a view of the server from the rear. The valid BDF mapping is as follows.

Let's take an example of when a 400G switch port is broken out into two 200G ports. The port `1/1/1` is split into `1/1/1:1` and `1/1/1:3`.

In the diagram above, `0000:84:00.0` is logically connected to `1/1/1:1`, and `0000:83:00.0` is logically connected to `1/1/1:3`.

For another example, `1/1/7:1` is logically connected to `0000:03:00.0`, and `1/1/7:3` is connected to `0000:04:00.0`.

Therefore, it is recommended to correct Figure 2 as follows:

**The leading way in each nic is the pf with revision id 0, and the numbering representing the NIC interface and the order of the revision id seem to be unrelated.**

- **NIC 1:** 0000:64:00.0 / 0000:63:00.0

- **NIC 2:** 0000:03:00.0 / 0000:04:00.0

- **NIC 3:** 0000:73:00.0 / 0000:74:00.0

- **NIC 4:** 0000:14:00.0 / 0000:13:00.0

- **NIC 5:** 0000:f3:00.0 / 0000:f4:00.0

- **NIC 6:** 0000:93:00.0 / 0000:94:00.0
