Overall hierarchy is as below

### IP_Core and interface ([file](https://github.mangoboost.io/MangoBoost/MB-Shell/blob/rdma/ips/sub/vendor_independent_wrapper/sub/ip_shell/sub/ip_core/fvsrc/IPCore_wrapper.sv))

IP_Core_wrapper integrates each IP’s IPCore module.

<div>

<div>

verilog

</div>

<div>

module IPCore_wrapper( axis_if.s s_axis00_BaseNIC_IPCore , axis_if.m m_axis01_IPCore_BaseNIC , axis_if.m m_axis02_IPCore_Ethernet , axis_if.s s_axis03_Ethernet_IPCore , axil_if.s s_axil07_IPBus_IPCore , axi_if.m m_axi00_IPCore_HPCIe , axi_if.s s_axi01_HPCIe_IPCore , axi_if.m m_axi02_IPCore_dram_ctrl , qdma_byp_out_if.s qdma_byp_out_HPCIe_IPCore , qdma_if.s qdma02_qdma_switch0_2x2_IPCore , mod_rst.s IPCore_mod_rst_IPCore , aclk.s aclk );

</div>

</div>

The interface is wrapped with system-verilog struct. Refer [this file](https://github.mangoboost.io/MangoBoost/MB-Shell/blob/rdma/ips/sub/common/fvsrc/mb_shell_common_svif.sv) to check components of each interface.

Explanations of each interface are as below

- s_axis00_BaseNIC_IPCore : packet from BaseNIC (BaseNIC’s TX)

- m_axis01_IPCore_BaseNIC : packet to BaseNIC (BaseNIC’s RX)

- m_axis02_IPCore_Ethernet : packet to MAC ( IPCore’s TX)

- s_axis03_Ethernet_IPCore : packet from MAC (IPCore’s RX)

- s_axil07_IPBus_IPCore : bar 2 AXI-lite port

- m_axi00_IPCore_HPCIe : master AXI from IPCore

- s_axi01_HPCIe_IPCore : bar 4 AXI-FULL port

- m_axi02_IPCore_dram_ctrl : axi interface to DRAM

- qdma_byp_out_HPCIe_IPCore : Not used.

- qdma02_qdma_switch0_2x2_IPCore : QDMA interface (c2h, h2c, usr_irq)

- IPCore_mod_rst_IPCore : reset for IPCore, generic reset module inside IPCore_wrapper using this reset.

- aclk : clock interface

So if there’s change on top interface , you need to fix [this file](https://github.mangoboost.io/MangoBoost/MB-Shell/blob/rdma/ips/sub/vendor_independent_wrapper/sub/ip_shell/sub/ip_core/sub/rdma/fvsrc/IPCore_rdma.sv)(IPCore_rdma.sv)

### How to add new files?

- If you add new modules on rdma_core/sysip/interface/onic , you don’t need to make new soft link because it’ll be automatically applied. ([link](https://github.mangoboost.io/MangoBoost/MB-Shell/tree/rdma/ips/sub/vendor_independent_wrapper/sub/ip_shell/sub/ip_core/sub/rdma/sub))

- If you add new verilog-base file, add new soft link on here

<div>

<div>

ln -s PATH/\* .

</div>

</div>

- If you changed something, you need to update dut.f

<div>

<div>

cd ips/sub/vendor_independent_wrapper ./generate_filelist.sh

</div>

</div>
