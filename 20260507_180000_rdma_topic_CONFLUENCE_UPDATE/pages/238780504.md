# Efficient User-Level Multi-Path Utilization in RDMA Networks

<!-- id=238780504 parent=238780418 -->

## Overview

Remote Direct Memory Access (RDMA) networks provide low-latency and high-throughput communication but suffer from inefficient multi-path utilization at the user level. Existing approaches rely on kernel-based load balancing, which introduces overheads and scalability issues. This paper presents a user-level multi-pathing mechanism that effectively distributes RDMA traffic without kernel involvement, reducing latency and improving throughput. The proposed approach dynamically adapts to congestion conditions and significantly outperforms conventional RDMA load-balancing schemes. The evaluation demonstrates substantial performance gains in various workloads, highlighting the importance of efficient user-level RDMA multi-pathing.

## Objective

The paper proposes a user-level multi-path RDMA utilization mechanism that improves throughput and reduces latency by dynamically distributing traffic across multiple RDMA paths without kernel intervention.

## Background

RDMA, Multi-path networking, Load balancing in high-speed networks, User-level networking stacks, Congestion control in RDMA

## As-Is

- RDMA networks offer high performance but lack efficient multi-path utilization at the user level.

- Existing solutions rely on kernel-based load balancing, which increases CPU utilization by up to 25%, introduces context-switching overhead, and fails to scale efficiently under high traffic conditions.

- Traditional RDMA congestion control mechanisms fail to dynamically adapt to varying network conditions, resulting in suboptimal load balancing, increased queueing delays, and inefficient bandwidth utilization.

- An ideal solution should be lightweight, adaptive, and capable of efficiently utilizing multiple paths without kernel involvement.

## To-Be

- The proposed solution implements a fully user-level approach to RDMA multi-path utilization, eliminating kernel overhead.

- It leverages a user-space path selection algorithm that continuously monitors real-time congestion metrics and dynamically redirects RDMA traffic, bypassing kernel-level processing to reduce overhead and improve efficiency.

- The mechanism integrates with existing RDMA libraries by modifying the connection management layer to enable multi-path capabilities, requiring only minor adjustments to application initialization routines.

## Methodology

- Implemented the user-level multi-path RDMA mechanism by extending the RDMA Verbs API with a custom load-balancing layer that dynamically selects optimal paths based on real-time congestion feedback.

- Used a mix of synthetic and real-world RDMA-based workloads, including microbenchmarks and distributed applications, to evaluate performance under varying congestion levels and network topologies.

- Conducted experiments on a testbed with multiple RDMA-enabled servers interconnected through a high-speed fabric.

- Measured latency, throughput, and CPU overhead to compare with existing RDMA load balancing methods.

- Compared performance against traditional kernel-based RDMA load balancing techniques using metrics such as throughput, latency, CPU overhead, and packet retransmissions in controlled congestion scenarios.

## Results

- The proposed user-level multi-path RDMA approach achieves higher throughput (up to 35% increase) and lower latency (up to 20% reduction) compared to kernel-based solutions.

- Experimental results show significant improvements in RDMA traffic distribution across multiple paths, leading to a 40% increase in link utilization.

- CPU overhead is reduced by 25% due to the elimination of kernel context switching.

- The system dynamically adapts to network congestion, improving resource utilization and reducing packet retransmissions by 30%.

- Future research directions include further optimization of dynamic path selection algorithms and extending the approach to broader RDMA workloads.
