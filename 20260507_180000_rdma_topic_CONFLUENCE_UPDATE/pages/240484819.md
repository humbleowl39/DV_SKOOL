# Fast Distributed Deep Learning over RDMA

<!-- id=240484819 parent=240550447 -->

## Overview

This paper argues that Remote Procedure Call (RPC), widely used in distributed deep learning frameworks like TensorFlow, is suboptimal when running over Remote Direct Memory Access (RDMA) networks. The study demonstrates that RPC incurs unnecessary memory copy overheads and fails to exploit the full potential of RDMA’s capabilities. Instead, the authors propose a lightweight RDMA-based communication mechanism that treats remote machines as “devices,” exposing simple memory operations for reading, writing, and allocation. This approach integrates with TensorFlow’s data-flow graph analyzer, enabling zero-copy tensor transfer. The proposed solution achieves up to 169% speed improvement over an optimized RPC-based RDMA implementation, leading to significantly faster training times for deep learning models.

## Objective

The paper aims to improve the efficiency of distributed deep learning over RDMA networks by eliminating unnecessary memory copies and leveraging direct memory access. The proposed approach achieves up to 169% speed improvement over an optimized RPC-based RDMA implementation in deep learning benchmarks.

## Background

Deep learning, Data-flow graphs, Remote Procedure Call (RPC), Remote Direct Memory Access (RDMA), High-performance computing (HPC) networking

## As-Is

- Current distributed deep learning frameworks, such as TensorFlow, use gRPC (a form of RPC) for inter-node communication, incurring overhead from data serialization and deserialization. Also, it requires memory copies between application buffers and RPC communication buffers, and introduces additional message fragmentation and reassembly overhead.

- RDMA enables direct memory access between machines, bypassing CPU intervention, and provides high bandwidth (e.g., 100Gbps) and low latency (~2µs round-trip time).

- TensorFlow’s existing RDMA implementation still relies on an RPC-based communication layer, introducing redundant memory copies between application buffers and RDMA buffers, which reduces the overall efficiency of direct memory access.

- Deep learning workloads primarily involve large, fixed-size tensors that exhibit predictable allocation and communication patterns within a data-flow graph. This predictability allows for efficient direct memory access, provided tensor locations are determined statically during compilation or runtime graph analysis.

## To-Be

- Treats remote nodes as “devices” with direct memory access interfaces, enabling direct RDMA-based communication that bypasses the CPU for remote data operations. It provides an API for allocating, reading, and writing remote memory, eliminating the need for intermediate buffers and reducing memory copy overhead, thereby improving the efficiency of distributed deep learning training.

- Use static analysis to predict tensor memory requirements, allowing for efficient preallocation of buffers, optimizing memory placement, and minimizing runtime memory fragmentation, which leads to improved computational efficiency and reduced communication latency.

- For static allocation, preallocate memory and transfer directly via one-sided RDMA write. For dynamic allocation, transfer metadata first, allocate remote buffers, and then transfer the actual data via RDMA read.

- Extends TensorFlow’s graph analyzer to determine tensor memory locations and optimize communication, identifies static vs. dynamic tensors, and preallocates memory accordingly.

## Methodology

- 8-server cluster with dual Intel Xeon CPUs, NVIDIA Tesla P100 GPUs, and 100Gbps InfiniBand RDMA network, running Ubuntu 16.04, CUDA 8.0, cuDNN 6.

- CNNs (AlexNet, VGGNet-16, Inception-v3), RNNs (LSTM, GRU), FCNs (FCN-5), and real-world workloads (Seq2Seq for translation, CIFAR-10 for image recognition, and sentence embedding (SE)).

- Performance Metrics: Mini-batch processing time, communication overhead, and model convergence speed.

- Comparison Baselines: gRPC over TCP, gRPC over RDMA, and RDMA with and without zero-copy optimization.

## Results

- Microbenchmark Performance: RDMA-based transfer achieves 1.7× to 61× improvement over gRPC.TCP. Compared to gRPC.RDMA, RDMA zero-copy achieves 1.3× to 14× improvement.

- Deep Learning Training Performance: Speedup over gRPC.RDMA includes 169% for AlexNet, 145% for VGGNet-16, and 151% for FCN-5. Zero-copy optimization reduces memory copy overhead by up to 21%.

- Model Convergence: Seq2Seq training time reduced from 220 minutes (gRPC.TCP) to 66 minutes (RDMA) (~3× speedup). CIFAR-10 training time reduced by 2.6× over gRPC.TCP.

- Scalability: RDMA-based approach scales well to 8 workers, achieving 5× to 7.9× speedups over single-node training. RDMA outperforms local execution beyond 2 workers.

- GPUDirect RDMA Gains: Further reduces training time by up to 54%, particularly benefiting communication-heavy models.
