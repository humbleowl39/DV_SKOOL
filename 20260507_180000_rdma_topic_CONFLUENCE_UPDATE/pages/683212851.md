# [SKRP-371] module list for bitwidth trimming

<!-- id=683212851 parent=7012616 -->

- 이 브랜치는 유진님의 [SKRP-371-example](https://github.mangoboost.io/yujin-chung/RDMA/commit/52e734d16a0d5ada0c9b6db8579de1c15ef5878b) 브랜치에서 우선 mb_struct.h 관련 내용만 선제적으로 따온 다음 부족하다고 생각되는 부분을 추가 + 이로 인해 재합성이 필요한 모든 모듈 싹 다 돌린 결과까지 포함하고 있습니다

- 개별 모듈 수정하실 때는 mb_struct.h에서 수정된 사항 + 유진님 커밋에서 먼저 시도되었던 부분 등을 레퍼런스로 삼되 이외에 좋은 아이디어가 있다면 얼마든지 시도해봐도 될 것 같아요

- 다음 7개 모듈은 mb_struct 바뀐 후 합성 자체가 실패했기 때문에 코드 수정 될때까지는 합성이 불가능해보여요 - wqe_decoder / requester_frontend / responder_prefetch / completer_frontend / frontend_memory / mtt_cache / rq_fetcher

- 합성에 실패한 모듈도 있고 해서 현재 이 브랜치가 완벽하게 검증된 건 아니라는 걸 감안해주시면 감사하겠습니다

- 본 커밋 전후로 달라진 전체 module standalone 결과

  - 전 - `/home/yoonhyuk.choi/timing/before`

  - 후 - `/home/yoonhyuk.choi/timing/after`
