# Overview

<!-- id=229998615 parent=229998593 -->

Priority-based flow control (PFC), IEEE standard 802.1Qbb, is a link-level flow control mechanism. PFC is intended to eliminate packet loss due to congestion on a network link. The flow control mechanism is similar to that used by IEEE 802.3x Ethernet PAUSE, but it operates on individual priorities. Instead of pausing all traffic on a link, PFC allows you to selectively pause traffic according to its class.

Operating at the data link layer, PFC assigns different priority levels to various types of traffic (we call this Traffic Class, TC). When congestion occurs, it can pause lower-priority traffic without affecting higher-priority flows. This mechanism is particularly beneficial in environments where multiple types of services share the same network infrastructure, as it helps maintain the performance and reliability of essential applications. In other words, this allows loss-sensitive protocols, such as Fibre Channel over Ethernet (FCoE), to coexist with traditional loss-insensitive protocols over the same unified fabric.

PFC uses the 802.3 PAUSE frame, and extends it with the following PFC fields:

- An 8-bit mask that specifies which [802.1p](https://en.wikipedia.org/wiki/IEEE_P802.1p) priority levels should be paused.

- A timer value (Quanta) for each priority specifying how long the traffic for that priority level should be paused.

When the receiver sends an 802.3 PAUSE frame with PFC data, the switch or NIC blocks the transmit of frames with the specified priority level to the port on which the receiver is connected. When the timer value expires, the switch resumes the transmit of paused frames on the port.

\+ It is worth noting that there are three primary QoS strategies managing demands for buffer memory and switch port bandwidth: dedicated switch buffers for an application, flow-control, and guaranteed media access for an application. Combining these three strategies enables a lossless Ethernet fabric for storage and other applications.
