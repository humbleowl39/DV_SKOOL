정답 글자·보기 문구·Bloom 태그·코드블록을 변경하지 않고, 기존 정답 내용 아래에 인과 교수형 산문 2~5문장을 추가한다.

**Why:** 기존 정답 내용을 재작성하면 검수 없이 정보가 달라질 위험이 있고, 정답 무변경 확인이 요건으로 명시됨.

**How to apply:** `??? answer` 블록 유무와 무관하게 동일 원칙 적용. 정답 항목(`**정답.**`)은 그대로 유지하고, `**Why.**` 항목을 단문에서 "왜 정답인지 + 핵심 오답이 왜 틀렸는지" 인과 산문 2~5문장으로 교체/확장한다. 파일이 12개처럼 많을 경우 Write 로 전체 파일을 재작성하는 것이 Edit 반복보다 효율적이며, 각 정답 블록을 일괄 검토해 정답 문자열 불변을 보장한다. 새 문항·보기·정답을 생성하지 않는다.
