## Q1. (Remember)

DP 의 두 조건은?

<details>
<summary>정답 / 해설</summary>

1. **Optimal Substructure** — 큰 문제의 최적해가 부분 문제의 최적해로 구성됨.
2. **Overlapping Subproblems** — 같은 부분 문제가 재귀 전개에서 여러 번 등장.

두 조건 중 하나라도 빠지면 DP 는 적용되지 않는다. Optimal Substructure 가 없으면 부분 문제의 최적해를 조합해도 전체 최적해가 보장되지 않아 DP 점화식 자체가 성립하지 않는다. Overlapping Subproblems 가 없으면 동일한 계산이 반복되지 않으므로 메모이제이션의 효과가 없고, 이 경우 단순 분할 정복으로 충분하다. 두 조건이 모두 성립할 때 비로소 DP 가 지수 복잡도를 다항 복잡도로 낮춰준다.

</details>
## Q2. (Understand)

Top-Down (memoization) 과 Bottom-Up (tabulation) 의 차이를 설명하라.

<details>
<summary>정답 / 해설</summary>

- **Top-Down** : 재귀 + cache. 자연스러운 사고 흐름이지만 함수 호출 오버헤드 있음.
- **Bottom-Up** : 작은 부분 문제부터 표를 채워나감. 함수 호출 ↓, 공간 최적화 가능.

수학적으론 동등. 실전에서는 자연스러운 쪽으로 작성 후 필요시 변환.

Top-Down 은 점화식을 그대로 코드로 옮기는 방식이라 처음 작성이 직관적이다. 단, 재귀 깊이가 깊으면 스택 오버플로우 위험이 있고 함수 호출 오버헤드가 쌓인다. Bottom-Up 은 가장 작은 기저 사례부터 테이블을 채워올라가므로 재귀가 없어 빠르고, 직전 두 개의 값만 필요한 문제(예: Fibonacci)에서는 테이블 전체를 저장하지 않고 변수 두 개로 공간을 O(N) 에서 O(1) 로 줄일 수 있다.

</details>
## Q3. (Apply)

Climbing Stairs (계단 N개, 한 번에 1 또는 2 step) 의 DP 점화식과 답을 적어라.

<details>
<summary>정답 / 해설</summary>

- State: `dp[i]` = i 번째 계단까지 도달 방법 수.
- 점화식: `dp[i] = dp[i-1] + dp[i-2]`.
- Base: `dp[0]=1, dp[1]=1`.
- 결과: dp[N] (Fibonacci 수열).
- 공간 최적화 → 두 변수만 유지 → O(1) 공간.

점화식이 Fibonacci 와 동일한 이유는 i 번째 계단에 도달하는 방법이 "한 칸 전(i-1) 에서 한 계단 밟기"와 "두 칸 전(i-2) 에서 두 계단 밟기" 두 가지뿐이기 때문이다. 따라서 경우의 수를 더하면 `dp[i-1] + dp[i-2]` 가 된다. Base case 를 `dp[0]=1` 로 잡는 이유는 "0번 계단에 이미 서 있다" 는 상태를 1가지로 정의해야 dp[1], dp[2] 가 올바르게 계산되기 때문이며, 이를 0으로 잡으면 모든 값이 0이 되는 오류가 생긴다.

</details>
## Q4. (Analyze)

DP state 정의가 잘못된 경우의 증상 3가지는?

<details>
<summary>정답 / 해설</summary>

1. **답이 달라짐 (틀림)** — state 가 부분 문제를 충분히 표현하지 못함 (정보 부족).
2. **무한 재귀 / 사이클** — state 변환이 단조 감소하지 않음.
3. **중복 / 과대 계산** — state 가 서로 의존성이 있는 차원을 놓쳐 같은 답을 여러 차원에서 셈.

해결: state 에 들어가야 할 차원을 "이 문제를 풀려면 무엇을 알아야 하는가" 로 다시 정의.

DP 의 오류는 대부분 state 설계 실패에서 온다. "답이 틀린다"는 증상은 state 가 같아도 실제로는 다른 상황을 구분하지 못하고 있다는 의미이므로, 어떤 정보가 누락되었는지 찾아 차원을 추가해야 한다. "무한 재귀"는 점화식에서 더 큰 state 를 호출하는 방향이 있다는 뜻으로, state 를 재정의해 항상 더 작은 문제로 축소되도록 만들어야 한다. "중복 계산"은 메모이제이션 키가 실제 계산 단위와 일치하지 않아 같은 결과를 여러 번 저장하는 경우다.

</details>
## Q5. (Evaluate)

같은 문제에 (a) DP (b) Greedy 가 모두 가능할 때 어느 것이 좋은가?

<details>
<summary>정답 / 해설</summary>

- **Greedy 가 정답이 증명되면** → Greedy. 더 빠르고 간결.
- **Greedy 가 반례가 있으면** → DP. 안전.

실무에서는 보통 **DP 가 default + 가능하면 Greedy 로 단순화**. 면접에서는 둘 다 알고 있다는 신호를 주면 점수 ↑.

Greedy 가 항상 DP 보다 빠르지만 더 좁은 조건에서만 통한다. Greedy 는 매 단계에서 지역적으로 최선인 선택이 전역 최선으로 이어진다는 것이 증명된 경우에만 올바른데, 이 증명이 어려우면 반례를 하나 찾아 DP 로 전환하는 것이 안전하다. 예를 들어 동전 교환 문제에서 동전 단위가 임의적이면 Greedy 는 틀릴 수 있지만, 단위가 1·5·10·25 처럼 배수 관계면 Greedy 가 최적임을 증명할 수 있다. 면접에서 이 구분을 말할 수 있으면 알고리즘 선택 능력을 보여주는 것이다.

</details>
