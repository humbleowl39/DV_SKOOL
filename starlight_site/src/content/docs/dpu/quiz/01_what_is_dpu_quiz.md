## Q1. (Remember)

spec이 제시한 DPU 도입의 세 가지 효과가 아닌 것은?

- [ ] A. CPU 자원 확보
- [ ] B. 성능 향상 (반복 작업 가속)
- [ ] C. 격리 강화
- [ ] D. 클럭 주파수 자동 상승

<details>
<summary>정답 / 해설</summary>

**D**. spec §1은 DPU가 인프라 작업을 별도 프로세서로 옮겨 (A) 호스트 CPU 자원 확보, (B) 패킷 처리·암호화 같은 반복 작업의 전용 하드웨어 가속, (C) 인프라 서비스의 격리 강화 — 세 효과를 제공한다고 설명합니다. 클럭 주파수 상승(D)은 DPU의 동기가 아니며 spec에 등장하지 않습니다.

</details>

## Q2. (Understand)

NIC, SmartNIC, DPU를 오프로드 범위가 좁은 것부터 넓은 순서로 배열하면?

<details>
<summary>정답 / 해설</summary>

**NIC → SmartNIC → DPU**. NIC는 체크섬·segmentation 같은 _고정_ 기능 오프로드만 제공하고, SmartNIC는 패킷 처리 로직을 _프로그래밍_ 할 수 있어 네트워크·보안 일부를 오프로드하며, DPU는 프로그래머블 코어·메모리·가속기를 갖춰 네트워크 외 _스토리지·보안까지 독립_ 처리합니다(spec §2). 이는 고정 오프로드 → 프로그래머블 오프로드 → 독립 인프라 프로세서라는 진화(§4)와 일치합니다.

</details>

## Q3. (Apply)

한 서버에서 vSwitch가 호스트 CPU 코어를 과도하게 소비하고 있다. DPU를 도입하면 이 작업은 어디서 처리되는가?

- [ ] A. 여전히 호스트 CPU (DPU는 스토리지만 담당)
- [ ] B. DPU의 데이터 패스 처리 엔진/프로그래머블 코어
- [ ] C. GPU
- [ ] D. 원격 스토리지 타깃

<details>
<summary>정답 / 해설</summary>

**B**. 가상 스위칭은 네트워킹 오프로드로, DPU 도입 시 패킷 분류·전달·정책이 DPU의 데이터 패스 처리 엔진과 프로그래머블 코어로 옮겨가 호스트 CPU 코어 소비가 줄어듭니다(본문 §3). A는 DPU가 네트워킹도 담당하므로 틀렸고, GPU(C)는 계산 워크로드용이며, 원격 스토리지(D)는 vSwitch와 무관합니다.

</details>

## Q4. (Apply)

GPU와 DPU의 공통점을 한 문장으로 적용해 설명하라.

<details>
<summary>정답 / 해설</summary>

둘 다 특정 워크로드를 호스트 CPU에서 분리해 전문 처리함으로써 **CPU가 애플리케이션 처리에 집중하도록 만든다**(spec §1.2). GPU는 그래픽·병렬 _계산_ 을, DPU는 네트워킹·스토리지·보안·가상화 같은 _인프라_ 작업을 분리합니다. 워크로드와 설계 목적은 다르지만 "CPU를 본업에 집중시킨다"는 목적은 같습니다.

</details>

## Q5. (Analyze)

두 제품이 각각 "DPU"와 "IPU"로 불린다. 둘을 비교할 때 명칭이 거의 의미 없는 이유를 분석하라.

<details>
<summary>정답 / 해설</summary>

DPU는 고정된 단일 하드웨어 규격이 아니며, 벤더마다 DPU·IPU 등 다른 이름을 쓰고 구현·제공 기능도 제품 목적에 따라 다릅니다(spec Overview, §6). 따라서 비교는 명칭이 아니라 (1) 어떤 네트워크/스토리지/보안/가상화 기능을 오프로드하는가, (2) 데이터 패스/제어 패스 중 어디까지 프로그래밍 가능한가, (3) 호스트와 어떻게 분리하는가로 해야 합니다. 같은 이름이라도 기능이 다르고, 다른 이름이라도 기능이 비슷할 수 있습니다.

</details>

## Q6. (Evaluate)

"DPU는 그냥 더 빠른 NIC"라는 주장을 평가하라.

<details>
<summary>정답 / 해설</summary>

부정확합니다. DPU는 대역폭만 높인 NIC가 아니라, 패킷 분류·정책 같은 _제어 로직_ 과 스토리지·보안 작업까지 호스트에서 가져와 _자체 프로그래머블 코어_ 에서 처리하고 호스트와 격리하는 독립 프로세서입니다(spec §2, §3). 외형이 PCIe 네트워크 카드라 NIC처럼 보이지만, 내부에 코어·메모리·가속기가 있고 오프로드 범위와 격리 능력이 NIC와 근본적으로 다릅니다. 따라서 "빠른 NIC"는 DPU의 본질(인프라 오프로드·격리)을 놓친 주장입니다.

</details>
