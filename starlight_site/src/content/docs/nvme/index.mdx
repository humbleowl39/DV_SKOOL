## 왜 NVMe / NVMe-oF를 배우는가

스토리지가 회전 디스크(HDD)에서 플래시 SSD로 넘어오면서, 정작 매체는 빨라졌는데 인터페이스가 발목을 잡는 상황이 생겼습니다. SATA/SAS는 본래 느린 HDD를 전제로 설계된 프로토콜이라 큐가 하나뿐이고 그 안에 명령을 최대 32개밖에 못 넣습니다. 멀티코어 CPU가 동시에 수만 건의 I/O를 쏟아내도 이 좁은 통로 하나로 줄을 세워야 하니, SSD의 잠재력이 그대로 묻혀 버립니다. NVMe(Non-Volatile Memory Express)는 이 병목을 없애기 위해 PCIe 위에서 처음부터 플래시를 겨냥해 다시 설계된 인터페이스입니다. 최대 64K개의 큐, 큐당 64K개의 명령이라는 압도적 병렬성으로 SSD를 CPU에 직접 붙입니다.

검증 엔지니어 입장에서 NVMe를 모르면 정작 필요한 순간에 막힙니다. host와 controller가 host 메모리의 ring buffer(SQ/CQ)와 doorbell 레지스터로 어떻게 명령을 주고받는지 설명하지 못하면, 시뮬레이션이 왜 멈췄는지(doorbell을 안 울렸나? CQ를 안 비웠나?)를 판단할 수 없습니다. 또한 NVMe-oF(NVMe over Fabrics, 원격 스토리지를 네트워크로 로컬처럼 쓰게 한 확장)가 등장하면서 스토리지 프로토콜이 네트워크 패브릭(RDMA — 원격 메모리를 CPU 없이 직접 읽고 쓰는 기술 / RoCEv2 — Ethernet 위에서 RDMA를 구현하는 방식) 위로 확장되었기 때문에, 이제는 NVMe 큐 한 쌍이 RDMA QP(Queue Pair, 송·수신 큐 한 쌍) 한 쌍에 어떻게 매핑되고 capsule이 RDMA SEND/RECV로 어떻게 운반되는지까지 알아야 원격 스토리지 IP(예: NRT, NVMe RDMA Target)를 검증할 수 있습니다. 이 코스는 그 흐름을 단계별로 풀어줍니다.

## 학습 목표

코스를 마치면 다음을 할 수 있습니다:

- **Explain (이해)** NVMe가 SATA/SAS 대비 병렬성·지연·CPU 오버헤드 측면에서 왜 유리한지, 그 차이가 어디서 오는지 설명
- **Trace (분석)** host가 SQ tail doorbell을 울리고 controller가 CQ에 CQE를 쓴 뒤 host가 CQ head doorbell로 응답을 소비하는 한 명령의 전 과정을 추적
- **Differentiate (분석)** Admin / IO / Fabric 세 커맨드 분류의 역할과, Fabric 커맨드(opcode 0x7F)가 NVMe-oF 환경에서만 쓰이는 이유를 구분
- **Apply (적용)** In-Capsule Data(ICD)와 Out-of-line(SGL + RDMA Read/Write) 데이터 전송 모델을 write/read 시나리오에 맞게 선택
- **Evaluate (평가)** NVMe-oF over RDMA 검증 환경에서 polling vs interrupt, capsule 크기, OoO completion 등 설계 결정을 근거 있게 판단

## 사전 지식

다음을 알고 있으면 본문이 매끄럽게 읽힙니다:

- **PCIe 기본**: NVMe controller는 PCIe 장치이므로 BAR, memory-mapped register, MSI/MSI-X 개념이 필요합니다. [PCIe 코스](../pcie/) 참고.
- **Ring buffer (circular queue)**: head/tail 포인터, wrap-around, empty/full 판정.
- **RDMA 기본** (4장에 필요): QP(Queue Pair), SEND/RECV, RDMA Read/Write, Memory Region. [RDMA 코스](../rdma/) 참고.
- **메모리-맵 IO**: 레지스터 offset과 doorbell write 의미.

## 학습 팁

:::tip[효율적 학습]
- **큐 모델을 손으로 그려라**: SQ/CQ ring buffer의 head/tail이 어떻게 움직이는지, doorbell이 어느 쪽을 갱신하는지 화이트보드로 그릴 수 있어야 2장 이후가 풀립니다.
- **PCIe를 먼저**: NVMe는 PCIe 위에 얹힙니다. BAR/MSI가 흐릿하면 [PCIe 코스](../pcie/)를 먼저 보세요.
- **4장은 RDMA와 함께**: NVMe-oF는 RDMA QP·capsule 개념이 전제입니다. [RDMA 코스](../rdma/)와 교차로 읽으면 이해가 빠릅니다.
:::

:::caution[흔한 함정]
- **Doorbell 누락**: SQE를 메모리에 써놓고 SQ tail doorbell을 안 울리면 controller는 명령이 온 줄 모릅니다 — 시뮬 hang.
- **CQ phase tag 오해**: polling 환경에서 CQE 유효성은 phase bit으로 판정합니다. head 포인터만 보면 안 됩니다(2장).
- **OoO completion**: 서로 다른 명령의 completion이 발행 순서와 다르게 올 수 있습니다 — scoreboard를 단일 큐로 짜면 오탐(4장).
:::
