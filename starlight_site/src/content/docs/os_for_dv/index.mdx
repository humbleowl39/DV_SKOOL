## 왜 HW/DV 엔지니어가 OS를 배워야 하는가

검증 엔지니어가 다루는 거의 모든 IP는 결국 OS가 운전합니다. DMA 엔진은 OS가 메모리에 만들어 둔 command block을 읽어 전송을 시작하고, 레지스터는 OS가 memory-mapped I/O로 write 하며, 인터럽트는 OS의 handler가 받아 처리합니다. NIC·스토리지 컨트롤러·MMU·IOMMU가 "맞게 동작한다"는 것은 곧 "OS가 기대하는 대로 동작한다"는 뜻입니다. 따라서 OS가 하드웨어를 *어떻게* 추상화하고 *무엇을* 요구하는지를 모르면, DUT의 동작이 버그인지 정상인지조차 판단하기 어렵습니다.

이 코스는 OS 교과서(Silberschatz *Operating System Concepts*, 10th ed.)의 핵심을 HW/DV 관점으로 다시 엮습니다. 막연히 "프로세스와 스케줄링"을 외우는 대신, dual-mode가 곧 protection ring이고, paging의 TLB가 곧 우리가 검증하는 MMU/IOMMU이며, DMA·MMIO·interrupt가 곧 우리 testbench가 자극하고 관찰하는 바로 그 메커니즘이라는 연결을 만듭니다. 이 연결이 생기면 spec을 읽을 때 "이 레지스터를 왜 이렇게 두었는가", "이 interrupt가 왜 이 시점에 필요한가"가 보이기 시작합니다.

## 학습 목표

코스를 마치면 다음을 할 수 있습니다:

- **Explain (이해)** OS의 두 역할(자원 관리·하드웨어 추상화)과 dual-mode/system call이 하드웨어 특권 분리에 어떻게 기대는지 설명
- **Differentiate (분석)** process/thread/context switch와 preemptive/nonpreemptive 스케줄링을, race condition을 부르는 원인과 함께 구분
- **Trace (분석)** logical→physical 주소 번역 경로(page table·TLB)와 DMA 전송 경로(command block·cycle stealing·완료 interrupt)를 단계별로 추적
- **Implement (적용)** memory-mapped I/O 기반 polling/interrupt handshake와 atomic 명령(test-and-set/CAS) 기반 lock의 동작을 코드로 구성
- **Evaluate (평가)** weakly-ordered 메모리 모델에서 memory barrier가 왜 필요한지, deadlock 네 조건 중 무엇을 깨야 하는지 판단
- **Design (생성)** least privilege·protection ring·access matrix 관점에서 device 격리(IOMMU) 정책을 설계

## 사전 지식

다음을 알고 있으면 본문이 매끄럽게 읽힙니다:

- **디지털 회로 기본**: 클럭, 동기 회로, register, FIFO, bus
- **C 언어 기본**: pointer, struct, load/store 개념
- **컴퓨터 구조 기초**: CPU·register·main memory·cache의 계층, instruction 실행 흐름

이 코스는 특정 제품과 무관한 일반 배경지식입니다. 더 깊은 하드웨어 토대는 컴퓨터 구조 자료를, 프로토콜 적용은 [AMBA 코스](../amba_protocols/)를 함께 보세요.

## 학습 팁

:::tip[효율적 학습]
- **HW로 환원하며 읽기**: 각 개념을 "이게 내가 검증하는 어떤 IP/레지스터/신호에 해당하나?"로 번역하며 읽으세요.
- **세 가지 제어 방식 비교**: polling vs interrupt vs DMA는 testbench 설계 사고의 핵심 — 표로 정리해 두세요.
- **번역 경로 그리기**: logical→TLB→page table→physical 경로를 손으로 그릴 수 있으면 MMU/IOMMU 검증이 쉬워집니다.
- **퀴즈 즉시**: 각 모듈 끝나면 바로 퀴즈를 풀고 막힌 절을 다시 읽으세요.
:::

:::caution[흔히 헷갈리는 지점]
DV 실무에서 가장 자주 혼동하는 것은 **interrupt vs polling 선택 기준**, **virtual vs physical 주소(DMA가 어느 쪽을 쓰는가)**, 그리고 **weakly-ordered 메모리에서 barrier가 없을 때 생기는 silent 버그**입니다. M04·M05에서 별도로 다룹니다.
:::
