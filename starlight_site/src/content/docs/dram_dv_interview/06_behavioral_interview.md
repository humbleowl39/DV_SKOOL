## 1. 지원동기 / 이직 사유 (★ 가장 민감)

**"왜 직전 회사를 1년 만에 떠나나"** — 반드시, 집요하게 나온다. 답변 골격:

- 합류 목적 = 신규 DPU IP를 from-scratch로 검증 + 자체 ASIC 양산 비즈니스 기여.
- 회사 방향이 **자체 ASIC 양산 → FPGA 기반 서버 솔루션 공급**으로 전환.
- 그 결과 ASIC tape-out·양산 sign-off 경험 축적이 어려워짐.
- "양산 sign-off까지 책임지는 DV 커리어"를 위해 이동 결심.
- **회사 비방 금지** — "회사의 합리적 전략 선택, 다만 내 커리어 목표와 갈림"으로.

**꼬리질문**
- *그럼 SK도 또 1년 만에 나가나?* → SK는 DRAM 양산 sign-off가 *핵심 업*이라 내 목표와 구조적으로 일치 → 오래 깊게 갈 환경. 잦은 이직이 아니라 "목표 정렬"의 결과.
- *전전 직장은 왜 나왔나?* → 다양한 SoC/IP를 경험했지만 대부분 기구축 환경 위 작업 → 방법론을 from-scratch로 체득하는 갈증 → 그래서 신규 IP from-scratch 환경으로 이동. (긍정 프레임)

## 2. 갈등 / 협업

- *동료/상사와 갈등과 해결?* → 상용 VIP 폐기 결정 시 팀 내 우려 → **데이터(메모리 점유율·crash율)로 설득 + thin VIP PoC로 증명**. "감정이 아니라 근거로 합의." 또는 설계자와 bug 판정 이견 → spec·C-model·파형 근거로 TB bug vs DUT bug 규명.
- *끝까지 안 맞으면?* → 의사결정 기준(일정·리스크·근거) 공유, 책임자 판단 존중, 결정 후엔 전력 실행.
- *타 부서(설계/PD) 협업의 어려움?* → 이기종 통합·deadline 사례. 커뮤니케이션·문서화로 해결.

## 3. 실패 / 좌절 / 스트레스

- *가장 큰 실패와 배운 점?* → 솔직한 실패 1개(예: 초기 가설 오판으로 시간 소모) + 그 후 프로세스 개선.
- *마감 압박 관리?* → 우선순위(리스크 큰 path 먼저) + 자동화로 반복 제거 + 일정 가시화.

## 4. 강점 / 약점 / 자기인식

- *강점?* → "통념을 의심하고 근본원인까지 파는 집요함" + "검증 자동화 시스템 구축력."
- *약점?* → 좁고 솔직하게 + 보완 노력. 예: 새 도메인 초반에 완벽주의로 속도가 느릴 때가 있어 의도적으로 MVP-먼저로 교정 중. **약점이 직무 핵심(DRAM 검증)을 부정하지 않도록** 주의.
- *본인을 한 문장으로?* → "통념을 깨는 검증 엔지니어" — 반드시 사례와 연결.

## 5. 포부 / 가치관 / 기타

- *입사 후 포부 / 5년 후?* → 단기 DRAM 검증 체계 습득·기여 → AI 검증 시스템 이식 → 차세대 메모리 검증 핵심 인력.
- *야근/주말이 잦다면?* → 양산 sign-off 책임의 무게 이해. 효율(자동화)로 줄이되 필요 시 몰입.
- *리더십 경험?* → Lead 다수(Secure Boot·MMU·UFS), 신규 환경 from-scratch 주도.
- *품질 vs 일정 딜레마?* → 리스크 기반 판단 + 투명한 보고. sign-off 기준은 타협하지 않는다.
- *마지막으로 하고 싶은 말?* → 핵심 메시지 1문장 재강조 + 입사 의지.
